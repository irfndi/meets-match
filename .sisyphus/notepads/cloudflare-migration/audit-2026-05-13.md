## Summary: Priority Ranked Gaps

### P0 — Blocks Core Functionality
1. **`GetPotentialMatches` algorithm** — Core matching feature. Bot `/match` command depends on it.
2. **`notification_delivery_attempts` table + migration** — Audit trail required.
3. **Bot webhook handler with grammy** — Bot cannot receive Telegram updates.

### P1 — Required for Feature Parity
4. **Notification retry with exponential backoff** — Without this, failed notifications are lost.
5. **Bot conversation handlers** — Profile setup flow.
6. **Worker cron triggers (re-engagement, DLQ health)** — Background jobs.
7. **Input validation on API router** — Security/robustness.

### P2 — Nice to Have
8. Missing D1 indexes (channel, idempotency, related_*)
9. `updateLastRemindedAt` endpoint
10. Sentry integration in TS workers
11. Config validation layer
12. Notification cleanup/expiry
