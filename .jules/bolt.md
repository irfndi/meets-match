## 2024-05-18 - Concurrent Effect.all for N+1 Query Resolution
**Learning:** In the `services/bot` codebase, when doing batch fetching of user profiles or similar data through grpc/effect logic, fetching sequentially within loops creates severe N+1 bottlenecks. By default `Effect.all` processes sequentially, so to achieve concurrency, `{ concurrency: 'unbounded' }` must be explicitly passed.
**Action:** When making multiple distinct fetches (e.g. up to 10 user profiles for a match list), always map the requests into an array and use `Effect.all(array, { concurrency: 'unbounded' })` to process them concurrently. Be sure to handle individual promise rejection natively using `.pipe(Effect.catchAll(() => Effect.succeed(null)))` before grouping so one failure doesn't reject the whole batch.
